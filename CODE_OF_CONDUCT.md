# Code of Conduct

## Our standard

We want `orya-eval` to be a professional, welcoming project for contributors of all experience levels.

Examples of behavior that help create that environment:

- being respectful and constructive in discussion
- assuming good intent while reviewing code and ideas
- giving actionable technical feedback
- accepting feedback gracefully
- focusing on what is best for the project and its users

Examples of unacceptable behavior:

- harassment, insults, or personal attacks
- discriminatory or exclusionary language
- trolling, bad-faith argument, or deliberate disruption
- publishing private information without permission

## Scope

This code of conduct applies in project spaces, including issues, pull requests, discussions, and other community interactions related to the repository.

## Enforcement

Project maintainers may remove, edit, or reject comments, commits, code, issues, and other contributions that do not align with this code of conduct.

## Reporting

If you experience or witness unacceptable behavior, open a private maintainer contact channel if one is available, or report it through the repository maintainers directly.
